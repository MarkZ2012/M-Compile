层 1 — 优化 pass（现在最值得做）
Conv-BN-ReLU 算子融合
把 BatchNorm 的权重折叠进 Conv 权重，消除独立 BN kernel 调用，减少内存读写往返
消除 batchnorm.c 的运行时开销
Buffer 复用 / 内存规划
当前 allocator 为每个中间结果静态分配。可以做生命周期分析，让已死亡的 buffer 在后续节点中复用，大幅压缩 peak 内存
嵌入式设备上效果最显著
常量折叠增强
现有 constant_fold.py 已有框架，可以扩展：把 BN 参数在编译期直接算入 Conv 偏置，Reshape/Flatten 在静态 shape 已知时直接消除
已有 pass_manager 框架
Conv 实现升级
现在 conv2d.c 是朴素实现。im2col + GEMM 的分解方式能充分利用缓存局部性；Winograd 对 3×3 kernel 有理论 2.25× 加速
ResNet18 推理速度直接受益
层 2 — 代码生成质量
Loop tiling / unrolling
在 emit_conv 等方法中生成带 tile 参数的循环体，让 L1/L2 cache 命中率大幅提升；unroll factor 可以写进 KernelSpec
generic 平台也能受益
OpenMP 多线程并行
在 emit_conv / emit_gemm 生成的外层循环加 #pragma omp parallel for，CMakeLists 加 find_package(OpenMP)，零硬件依赖地利用多核
服务器场景立竿见影
权重 mmap 懒加载
目前 _load() 在 init 时把全部权重读进内存。改用 mmap 后 OS 按需换页，首次推理延迟下降，冷启动内存占用减半
改 runtime/src/loader.c
INT8 量化权重
权重存 int8 二进制，加载时反量化或直接用 int8 kernel 计算。模型体积压缩 4×，嵌入式 flash 占用大幅下降
需要新增 quantize pass
层 3 — 编译器基础设施
动态 shape 支持
现在 shape_infer 要求编译期已知所有维度。加入符号维度（如 batch=N）后，生成的 C 函数接受运行时 shape 参数，支持变长输入
graph_ir.py + shape_infer.py
Graph 可视化 / 调试器
在 optimizer 之前和之后各 dump 一次 Graphviz DOT 文件，直观看到哪些节点被融合/消除，大幅降低 pass 调试成本
加一个 tools/viz.py 即可
数值精度回归测试
用 PyTorch 跑同一张图的输出作为 golden，与生成 C 代码的输出做 cosine similarity 对比，自动化验证每次优化不破坏精度
tools/test/ 已有框架可扩展
算子覆盖率扩展
目前支持 ResNet 系列够用，但 MobileNet 需要 DepthwiseConv，BERT 需要 LayerNorm / Attention。按优先级排队实现
DepthwiseConv 优先级最高
层 4 — 长期演进
Auto-tuning
对 tile size、unroll factor 等超参数做搜索（类似 TVM 的 AutoScheduler），在目标机器上测速后选最优配置写入 KernelSpec
需要 benchmarking 基础设施
算子间流水线并行
分析节点依赖图，把无数据依赖的子图分配到不同线程并行执行，而不是现在的严格串行遍历
需要任务图调度器
